## 你居然还在手动注入？

太愚蠢了，太费力了，太慢了！

我还是直接教你sqlmap，**SQL注入一把梭哈**，享受极致的注入体验

## 安装

GitHub下载zip即可，Kali自带

sqlmap基于python实现，使用命令`python sqlmap.py <参数>`启动

## 使用

### 参数表

#### 目标

| 参数名        | 含义                                                | 适用场景                            |
| ------------- | --------------------------------------------------- | ----------------------------------- |
| -u, --url     | 指定url                                             | 直接跟链接或IP                      |
| -m, --bulk    | 批量扫描多个目标，后面跟一个包含多个url的txt文件    | 批量找漏洞（信息搜集）              |
| -r, --request | 从文本中加载http请求，抓包保存为txt后使用此参数加载 | 抓HTTP报文并保存为txt文件使用       |
| --data        | 指定POST数据                                        | 简单POST注入，例如`id=1&name=admin` |

#### 指定注入点

| 参数名          | 含义             | 适用场景                                                     |
| --------------- | ---------------- | ------------------------------------------------------------ |
| -p, --param     | 指定测试的参数名 | 例如`-p id`，指定参数`id`                                    |
| --skip          | 跳过的参数名     | 例如`--skip=token`，跳过`token`防止动态变化                  |
| --dbms          | 猜测数据库类型   | 例如`dbms=mysql`                                             |
| -t, --technique | 指定注入方式     | 默认全开<br />布尔盲注`B`，报错`E`，联合查询`U`，堆叠注入`S`，时间盲注`T` |

#### 数据提取（脱库）

| 参数           | 含义                   | 备注                                   |
| -------------- | ---------------------- | -------------------------------------- |
| -D, -T, -C     | 指定库、表、列（字段） | 一般配合下列参数使用                   |
| --current-user | 获取当前数据库用户     | 不需要值                               |
| --current-db   | 获取当前数据库名       | 不需要值                               |
| --dbs          | 列出数据库名           | 不需要值                               |
| --tables       | 列出指定库的所有表     | 必须配合`-D`使用                       |
| --columns      | 列出指定表的所有字段   | 必须配合`-D`和`-T`使用                 |
| --dump         | 直接输出表内所有内容   | 最常用的参数，脱库的终极目的，不需要值 |
| --count        | 输出行数               | 不需要值，靠行数可以判断表内是否有数据 |
| --dump-all     | 输出所有库的所有数据   | 不常用，意义不大，不需要值             |

#### 绕过

| 参数                | 含义             | 备注                                                         |
| ------------------- | ---------------- | ------------------------------------------------------------ |
| --level             | 测试级别（1-5）  | 默认 1，若注入点在Cookie/UA里，需要调高2或3。题目做不出时可以试试调大 |
| --risk              | 风险等级（1-3）  | 默认 1。调高会使用更危险的payload，可能破坏数据，慎用        |
| --tamper            | 使用脚本绕过WAF  | 例如`--tamper=space2comment`（空格变注释），可以使用逗号分隔多个脚本 |
| --random-agent      | 随机更换UA       | 防止UA过滤，不需要值                                         |
| --delay             | 请求间隔时间     | 防止触发频率限制                                             |
| --threads           | 多线程数量       | 默认 1                                                       |
| --prefix / --suffix | 自定义注入前后缀 | 遇到特殊题目，如`'`过滤、`)`闭合时使用                       |

#### 操作系统与文件读取

| 参数         | 含义                 | 适用场景                        |
| ------------ | -------------------- | ------------------------------- |
| --os-shell   | 尝试获取shell        | 不需要值                        |
| --os-cmd     | 尝试执行单条系统命令 | 例如`--os-cmd="whoami"`         |
| --file-read  | 尝试读取文件         | 例如`--file-read="/etc/passwd"` |
| --file-write | 尝试上传文件         | 常用来上传backdoor或webshell    |
| --file-dest  | 指定上传路径         | 配合`--file-write`使用          |

#### 其他小细节

`--batch`，自动确认（按Y回车），解放双手

`*`，指定注入位置，例如`page=*&id=1`，指定注入的参数是`page`，与参数`-p`作用一致
